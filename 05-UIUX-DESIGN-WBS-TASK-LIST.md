## NOTES

- All task hour estimates assume a single designer. Parallel work across 2–3 designers can compress the timeline.
- Phase 7 tasks are iterative — expect multiple rounds of refinement based on testing feedback.
- Native mobile app designs (Phase 5) should reference web mobile responsive designs for consistency but adapt to platform-specific patterns (iOS Human Interface Guidelines, Material Design 3).
- Admin dashboard (Phase 6) shares the design system with the user app but uses a distinct layout and darker sidebar to differentiate the admin experience.
- Email template designs (ENH-012) should be tested across major email clients (Gmail, Outlook, Apple Mail, Yahoo) for rendering consistency.
- All designs must account for the 12 uploadable artifact types defined in the PRD (Section 9) and their placement on generated documents.
- The 18 document type forms (WEB-032 to WEB-049) share a common layout pattern — design the first (Writ of Summons) as the master template, then adapt for each type.
- Figma auto-layout and component variants should be used extensively to maintain consistency and enable efficient updates.
